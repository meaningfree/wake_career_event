# 施策立案ライブデモ プロンプト集

コーディングエージェント（Cursor、Claude Code、Amazon Q Developer など）を活用した不動産ポータル向け施策立案ワークフローのデモ用プロンプト集です。

## 概要

このプロジェクトは、AIエージェントと対話しながら施策立案を行うプロセスを体験するためのデモ教材です。マーケットリサーチからPRD（Product Requirements Document）作成まで、一連の施策立案プロセスを8つのステップで実践できます。

## 対象者

- プロダクトマネージャー
- 企画担当者
- AIを活用した業務効率化に興味のある方

## ファイル構成

```
wake_career_event/
├── README.md           # このファイル
├── prompts.md          # デモで使用するプロンプト集
├── data/               # サンプルデータ
│   ├── prd_sample_01.md〜05.md   # PRDサンプル（テンプレート生成用）
│   ├── executed_feature_spec.md  # 実施済み施策の仕様書
│   └── executed_feature_result.md # 効果測定結果
└── output/             # AIが生成するアウトプット（実行時に作成）
```

## ワークフロー

| ステップ | 内容 | 出力ファイル |
|----------|------|--------------|
| 1. マーケットリサーチ | 競合サービスのユーザー層分析 | `output/01_market_research.md` |
| 2. ペルソナ生成 | ターゲットセグメントからペルソナを作成 | `output/02_persona.md` |
| 3. 施策アイデア出し | 課題×解決策の組み合わせを提案 | `output/03_selected_idea.md` |
| 4. 施策改善アイデア出し（別ルート） | 既存施策の分析と改善案 | `output/04_selected_idea_retry.md` |
| 5. 専門家フィードバック | 複数の専門家視点でレビュー | `output/05_feedback_log.md` |
| 6. 分析ストーリー生成 | 施策の根拠となる仮説を作成 | `output/06_analysis_story.md` |
| 7. PRDテンプレート作成 | サンプルからテンプレートを抽出 | `output/07_prd_template.md` |
| 8. PRD作成 | 最終成果物としてPRDを作成 | `output/08_prd.md` |

## 使い方

### 対応ツール

以下のコーディングエージェントで利用できます：

- **Cursor** - Agent モード
- **Claude Code** - Anthropic 製 CLI エージェント
- **Codex** - OpenAI 製 CLI エージェント
- **Amazon Q Developer** - AWS 製エージェント
- その他、ファイル読み書きが可能なAIエージェント

### 準備

1. お使いのコーディングエージェントで本プロジェクトを開く
2. `prompts.md` を開く

### 実行手順

1. `prompts.md` から使用したいプロンプトをコピー
2. エージェントモードでプロンプトを実行
3. 必要に応じてプレースホルダー（`[競合サービス名]` など）を具体的な値に置き換える
4. AIの出力を確認し、対話を続ける

### プロンプトの流れ

```
新規施策立案ルート:
1 → 2 → 3 → 5 → 6 → 7 → 8

既存施策改善ルート:
4 → 5 → 6 → 7 → 8
```

## 注意事項

- このデモは教育・体験目的です
- 出力されるアイデアやPRDはサンプルであり、実際の施策として使用する場合は別途検討が必要です
- AIの出力内容は実行ごとに異なる場合があります
